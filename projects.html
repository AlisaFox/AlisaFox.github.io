A majority of my more complicated projects were built in C. For example, I've made a "tiny shell" that is a simplified version of a bash terminal. It also has unique commands like history, which would show the last 100 user prompts; or limit which would limit the amount of memory the pseudo-shell would be allowed to use.

Another one of my projects was a simple file system. It mostly involved two parts: organizing the simulated disk into components like the superblock and inodes. Then the next part was making functions to make files, write and read from them, and so on. Because this is a simplefied file system, making subdirectories is not yet supported.

Java is the language I am familiar with the most. I've made a lot of small projects. Some of them are games: like tic-tac-toe against a bot with a twist, the board size was specified by the user. So if you ever wanted to know what it is like to play tic-tac-toe on a 10 by 10 board, now you can!

In this warehouse representation I delved into the object-oriented aspect of java. There are boxes of different sizes, that have to fit on shelves in a warehouse in an optimal way. I used the merge-sort algorithm, to reorganize the boxes and the shelves according to their heights. I have also played with classes a bit, for example urgent box is a subclass of box, which can be added to a separate shipping list so that it leaves the warehouse faster.
